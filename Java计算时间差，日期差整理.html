3、Duration

期间

JDK1.8 工具：==这个类是不可变的和线程安全的==

计算两个时间之间的： 纳秒，毫秒，秒，分，小时，天 之间的差。

注意： 不能计算月数，年数的差

此类以秒和纳秒为单位对时间量或时间量进行建模。 可以使用其他基于持续时间的单位来访问它，例如分钟和小时。 此外，可以使用DAYS单位并将其视为正好等于 24 小时，从而忽略夏令时的影响。 有关此类的基于日期的等效项，请参见Period 。

物理持续时间可以是无限长的。 为实用起见，持续时间存储有类似于Instant的约束。 持续时间使用纳秒分辨率，最大值为可以保存在long中的秒数。 这大于目前估计的宇宙年龄。
持续时间的范围需要存储一个大于long的数字。

为了实现这一点，该类存储了一个表示秒的long和一个表示纳秒的int ，它始终在 0 到 999,999,999 之间。 该模型具有定向持续时间，这意味着持续时间可能为负数。
持续时间以“秒”为单位，但不一定与基于原子钟的科学“SI 秒”定义相同。 这种差异仅影响在闰秒附近测量的持续时间，不应影响大多数应用程序。 有关第二个和时间尺度的含义的讨论，请参见Instant 。
这是一个基于值的类；

在Duration的实例上使用身份敏感操作（包括引用相等 ( == )、身份哈希码或同步）可能会产生不可预知的结果，应该避免。 应该使用equals方法进行比较。

4、ChronoUnit

JDK1.8 工具

可以处理时间差。单位：小打 Nanos(纳秒)，大到Forever (永远) 都可以处理。

一组标准的日期期间单位。
这组单元提供基于单元的访问来操作日期、时间或日期时间。 可以通过实现TemporalUnit来扩展标准的单位集。
这些单位旨在适用于多个日历系统。 例如，大多数非 ISO 日历系统定义了年、月和日的单位，只是规则略有不同。 每个单元的文档解释了它是如何运作的。

实施要求：
这是一个最终的、不可变的和线程安全的枚举
