    ​ 初始状态

    方式一： rebase

    原理： git -i rebase 上一次的commit_id.

    遇到冲突时：

    ​

    操作： 右键–>drop commit

    注意： 提交内容丢失，不在工作区；提交日志同样没有.删除中间某次commit 需要手动merge ，然后再继续rebase

    方式二： reset

    原理： git reset –soft 上一次的commit_id

    操作： 选择删除的commit （此commit 需要是最新的），右键–> Undo Commit

    注意： 提交日志消失，change 内容没有删除，在工作区，暂存区

    ​ 点击OK

    场景： 删除最新的一次commit 或 多次连续的commit

    操作： 选择要删除commit前的 commit ，如下图所示，右键–> “reset current branch to here”

    Soft

    保留工作区、暂存区

    ​

    Mixed

    保留工作区，清除暂存区

    Hard

    ​ 此次commit之后的提交内容移除，并且日志删除。 测地清除暂存区、工作区

    Keep

    未接触，暂不整理

    Hard Mixed Soft 影响的范围

    方式三： revert

    git revert 之前的提交仍会保留在 git log 中，而此次撤销会做为一次新的提交

    原理： git revert 要恢复的commit_id

    操作： 1右键–> 2

    注意： 当前commit记录会保留，revert 后会做一次新的提交。 当没有冲突时 顺利 revert ，有冲突时，需要手动 merge

    总结

    1. 删除最新的本地commit ，使用3中方式都行

    2. 删除中间的某次提交，建议使用方式三，团队开发时，不容易出现问题

    3. 删除一直到最新commit的多次commit，建议使用方式一

    4. 删除中间某次commit误删了，可以使用

    ​ 使用修改最新的commit (直接命令行运行）： git commit –amend

    ​ 修改中间的某次的commit： git rebase -i hash值（某次commit前一次的hash值）
